深入理解Objective-C中的KVC
2019-09-02
字数总计: 1.1k|阅读时长: 4 分钟

KeyValueCoding 键值编码模式，为我们提供使用 key 来访问和赋值对象属性的一种方式；利用Objective-C中的Runtime机制实现的；它是我们实现 KVO 的基础。

通过上面小实例，我们简单的了解如何使用 KeyValueCoding 来访问和赋值对象的属性；下面我们来介绍下常用 KeyValueCoding 的方法。

赋值相关：

  • - (void)setValue:(nullable id)value forKey:(NSString *)key;
  • - (void)setValue:(nullable id)value forKeyPath:(NSString *)keyPath;
  • - (void)setValue:(nullable id)value forUndefinedKey:(NSString *)key;
  • - (void)setNilValueForKey:(NSString *)key;
  • - (void)setValuesForKeysWithDictionary:(NSDictionary<NSString *, id> *)keyedValues;

取值相关：

  • - (nullable id)valueForKey:(NSString *)key;
  • - (nullable id)valueForKeyPath:(NSString *)keyPath;
  • - (nullable id)valueForUndefinedKey:(NSString *)key;
  • - (NSDictionary<NSString *, id> *)dictionaryWithValuesForKeys:(NSArray<NSString *> *)keys;

其他方法：

  • + (BOOL)accessInstanceVariablesDirectly;: 判断是否可以访问对象的成员变量，默认为YES；
  • - (BOOL)validateValue:(inout id _Nullable * _Nonnull)ioValue forKey:(NSString *)inKey error:(out NSError **)outError;: 用来验证 ioVale 是否可以保存到 inKey 中去；需要验证时，可以重写该方法；默认返回YES；
  • - (BOOL)validateValue:(inout id _Nullable * _Nonnull)ioValue forKeyPath:(NSString *)inKeyPath error:(out NSError **)outError

这些方法都在 NSObject 中的 NSKeyValueCoding 的分类中，苹果处理为我们提供了 NSObject 的 NSKeyValueCoding 分类外，NSArray、NSDictionary、NSMutableDictionary、NSOrderedSet、NSSet都会有 NSKeyValueCoding 分类。

KeyValueCoding 赋值时，如何寻找”Key”并赋值?

  1. 先寻找是否有 setKey 方法
  2. 调用+ (BOOL)accessInstanceVariablesDirectly类方法；若返回NO，直接执行步骤6；若返回YES，继续执行下面步骤，
  3. 寻找”_key”成员变量；若有，则直接赋值结束
  4. 寻找”_isKey”成员变量；若有，则直接赋值结束
  5. 寻找”isKey”成员变量；若有，则直接赋值结束
  6. 调用- (void)setValue:(id)value forUndefinedKey:(NSString *)key后直接结束；若没有该方法，则crash。

KeyValueCoding 取值时，如何寻找”Key”取值？

  1. 先按照顺序寻找是否有 getKey、key、isKey、_key 中的任意一个方法，若有则直接调用并结束
  2. 若没有找到步骤1中的任意一个方法，则会寻找 countOf、objectInKeyAtIndexes或者keyAtIndexes
  3. 若还是没有找到步骤2中的方法；则会继续寻找 countOf、enumeratorOf、memberOf
  4. 若没有找到步骤3中的方法，调用+ (BOOL)accessInstanceVariablesDirectly类方法；若返回NO，直接执行步骤6；若返回YES，继续执行下面步骤，
  5. 按照 _key、_isKey、key、isKey 的顺序寻找成员变量是否存在，若存在则直接获取值并结束
  6. 上述步骤中的方法或者成员变量都没有找到，则会调用 - (id)valueForUndefinedKey:(NSString *)key 方法，若没有实现该方法，则会 Crash。
